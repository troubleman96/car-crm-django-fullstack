{% comment %} ================================================================================ TEMPLATE: chatbot/widget.html PURPOSE: Floating chatbot widget that appears on every page via ``include 'chatbot/widget.html'``. Features: - Toggle open/close via a circular floating button. - Session-based conversation history. - Sends user messages to a Django backend endpoint via fetch(). - Displays bot replies in a scrollable chat window. - Pre-populates the user's phone number from Django's auth user context (if logged in). ARCHITECTURE OVERVIEW: ┌─────────────────────────────────────────────────────────┐ │ widget.html (this file) │ │ ┌───────────────────────────────────────────────────┐ │ │ │ Alpine.js component: x-data="chatWidget()" │ │ │ │ Manages open/close, messages array, session ID, │ │ │ │ phone, and the send()/toggle()/scrollDown() │ │ │ │ methods. │ │ │ └───────────────────────────────────────────────────┘ │ │ ┌───────────────────────────────────────────────────┐ │ │ │ │ │ │ │ Standalone function defining the Alpine store. │ │ │ │ The fetch() POSTs JSON to /chatbot/message/. │ │ │ └───────────────────────────────────────────────────┘ │ └─────────────────────────────────────────────────────────┘ The Django view at /chatbot/message/ receives: { "message": "...", "session_id": null|"abc", "phone": "..." } And returns: { "reply": "...", "session_id": "abc" } The session_id is stored client-side and passed with every subsequent message so the backend can maintain conversation continuity. ============================================================================ {% endcomment %} {% comment %} =========================================================================== ALPINE.JS ROOT ELEMENT --------------------------------------------------------------- x-data="chatWidget()" -> initialises the component with the state and methods returned by the chatWidget() function. x-init="init()" -> runs the init() method once when the component mounts. {% comment %} =========================================================================== ALPINE.JS CDN --------------------------------------------------------------- This loads the Alpine.js library from a CDN. The defer attribute ensures the script is fetched in parallel with the page and executed after the HTML has been parsed. Alpine processes all x- attributes and x-data components on DOMContentLoaded. IMPORTANT FOR STUDENTS: In production you would typically install Alpine via npm (or download it) rather than relying on an external CDN, to avoid third-party downtime and potential security risks from CDN compromise. =========================================================================== {% endcomment %} {% comment %} =========================================================================== CHAT WIDGET ALPINE.JS COMPONENT -- chatWidget() --------------------------------------------------------------- This is a plain JavaScript function that returns an object. Alpine.js picks it up from the global scope (window) and uses it as the data model for any element with x-data="chatWidget()". PROPERTIES: open : Boolean -- whether the chat window is visible. input : String -- the current value of the message input field. sessionId : String|null -- a session token returned by the backend. Used to maintain conversation continuity across messages. Starts as null; after the first response the backend assigns a session_id that we store and send with every subsequent message. phone : String -- the user's phone number, pre-populated from the Django auth system if available. messages : Array<{id: Number, sender: String, message: String}> The conversation history. Pre-loaded with a welcome message from the bot. METHODS: toggle() : Opens/closes the chat. Auto-scrolls to the bottom when opening so the user sees the latest messages. send() : (async) Sends the current input to the backend API via fetch(), appends the user message to the UI immediately (optimistic update), then appends the bot's reply when the response arrives. scrollDown() : Scrolls the messages container (#chatMessages) to its full scroll height so new messages are visible. init() : Lifecycle hook that runs on component initialisation. Pre-populates phone with the logged-in user's phone number from the Django template context (if available). fetch() API CALL (in send()): POST /chatbot/message/ Headers: Content-Type: application/json X-CSRFToken: csrf_token -- Django's CSRF protection token. This is rendered by the template so it is available to JavaScript. The Django view uses CsrfViewMiddleware to verify this token on every POST. Body: { "message": "", "session_id": "", "phone": "" } Response (JSON): { "reply": "", "session_id": "" } =========================================================================== {% endcomment %}
